📊 Stock Analysis App

Architecture overview and run guide

Overview

Scalable stock analysis application with FastAPI backend and Streamlit frontend. Computes annual return, volatility, Value at Risk (VaR), RSI, and MACD for a given ticker.

Architecture

Layered design with clear separation of concerns:

project_root/
├── app/
│   ├── main.py              # FastAPI app entry point
│   ├── domain/
│   │   ├── models.py        # StockMetrics, TickerRequest
│   │   ├── entities.py      # StockPriceData
│   │   └── interfaces.py    # IStockRepository, IStockAnalysisService
│   ├── services/
│   │   └── stock_analysis_service.py   # Business logic (metrics computation)
│   ├── repositories/
│   │   └── stock_repository.py        # Data fetching (Yahoo Finance)
│   ├── api/
│   │   └── routes.py       # REST endpoints
│   └── utils/
│       └── logger.py       # Logging (file + console)
├── streamlit_app.py        # Frontend (input + display only)
├── tests/
│   ├── test_models.py
│   ├── test_services.py
│   └── test_api.py
├── requirements.txt
├── pyproject.toml
└── README.md

Layer Responsibilities

LayerPurpose
DomainModels, entities, and interfaces (contracts)
RepositoriesFetch stock data from Yahoo Finance via yfinance
ServicesCompute metrics: annual return, volatility, VaR, RSI, MACD
APIFastAPI routes exposing sync endpoints for single/batch analysis
FrontendStreamlit UI for input and display; no business logic

How to Run

1. Install dependencies

cd project_root
pip install -r requirements.txt

2. Start the FastAPI backend

uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

API runs at http://localhost:8000. Docs at http://localhost:8000/docs.

3. Start the Streamlit frontend

streamlit run streamlit_app.py

UI runs at http://localhost:8501. Enter a ticker and click Submit.

4. Run tests

pytest tests/ -v
